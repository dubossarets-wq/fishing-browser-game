# Ассеты, которые сейчас нарисованы процедурно (нужны текстуры)

Сейчас вся сцена и часть UI рисуются кодом (градиенты/примитивы Canvas 2D, эмодзи в интерфейсе) —
реальных изображений в проекте нет вообще (`public/` пустой, только `favicon.svg` из шаблона Vite).
Ниже — всё, что можно заменить на настоящие текстуры/спрайты, по приоритету.

Формат/движок: Canvas 2D, слои рисуются как `drawImage()`, поддержит PNG (с альфа-каналом) или WebP.
Все размеры — ориентировочные, сцена адаптивная (canvas растягивается на весь контейнер).

---

## 1. Фон локации (самое заметное) — высокий приоритет

Для каждой из 3 локаций (`src/data/locations/locations.ts`) сейчас процедурно рисуются градиент неба,
силуэт гор, ёлки-треугольники и вода-градиент. Вместо этого нужен набор слоёв (parallax) на локацию:

| Слой | Что сейчас | Нужно |
|---|---|---|
| Небо (день/закат/ночь) | цветовой градиент, плавно интерполируется по часам | можно оставить процедурным (градиент дешевле и уже плавно анимирован) или дать 2-3 фоновые текстуры на смену |
| Дальние горы/скалы | залитый полигон с синусоидой | PNG-силуэт, ширина тянется на всю сцену (tileable по X), прозрачный верх |
| Ближние горы/берег | то же | то же, второй параллакс-слой |
| Линия деревьев | треугольники-ёлки | PNG-полоса леса, tileable по X, ~200×80px паттерн |
| Вода | градиент + синусоидальная рябь | текстура воды (можно оставить рябь как оверлей поверх текстуры) |
| Берег/причал (передний план) | залитый прямоугольник + примитивный держатель для удочки | PNG земли/настила с реальной фактурой дерева/гальки |

Нужно на локацию: **clay_bend** (глинистый обрыв), **stone_reservoir** (каменистое водохранилище),
**quiet_backwater** (заросшая старица) — три разных набора слоёв, палитры уже заданы в
`src/engine/canvas/sceneTheme.ts` (LOCATION_PALETTES) как ориентир по цвету.

## 2. Снасть и рыба на сцене

| Ассет | Где | Сейчас | Нужно |
|---|---|---|---|
| Удочка (3 шт) | `FishingCanvas.tsx` | линия + дуга (`quadraticCurveTo`) | PNG-спрайт удилища, поворачивается на canvas |
| Держатель удочки | там же | 3 линии | маленький спрайт рогатки/держателя |
| Поплавок | там же | 2 кружка (красный+белый) | спрайт поплавка 8–12px |
| Пойманная рыба (подвес на леске) | `drawCaughtFish` | обобщённый серый силуэт рыбы для ЛЮБОГО вида | **10 отдельных спрайтов рыб** — по одному на вид (см. список ниже), не только силуэт, а узнаваемый вид |
| Всплеск/рябь от заброса и поклёвки | процедурные концентрические круги | спрайт-анимация всплеска (2-4 кадра) или частицы |

### 10 видов рыб, которым нужен отдельный спрайт (`src/data/fish/species.ts`)
Карась, Плотва, Краснопёрка, Лещ, Линь, Окунь, Щука, Судак, Сом, Карп —
сейчас все визуально идентичны (один и тот же серый овал), хотя механически полностью различны.
Желательно 2 варианта на вид (обычный/крупный трофейный) или хотя бы силуэт разного размера/окраса.

## 4. Иконки предметов снастей — есть только текст, нет вообще ничего

`src/game/equipment/types.ts` — типы предметов (`RodItem`, `ReelItem`, `LineItem`, `HookItem`,
`SinkerItem`, `FloatItem`, `FeederItem`, `BaitItem`, `GroundbaitItem`, `FoodItem`) **не имеют поля под
иконку вообще** — в магазине и инвентаре сейчас только название текстом. Нужно:

- по 1 иконке на категорию как минимум (10 иконок: удилище/катушка/леска/крючок/грузило/поплавок/
  кормушка/прикормка/еда + отдельно на каждую наживку — червь, опарыш, мотыль, кукуруза, хлеб, тесто,
  горох, живец, бойл, пеллетс, воблер, блесна, силикон, джиг-головка — **14 видов наживок/приманок**)
- в идеале — иконка на каждый конкретный предмет (6 удилищ, 3 катушки, 4 лески и т.д.), но это уже
  желательно, не критично для читаемости магазина

Если добавите иконки — понадобится добавить поле `icon?: string` в типы предметов и прокинуть в UI
(`ShopTab` в `BaseModal.tsx`, `SlotRow` в `RodSetupModal.tsx`) — это уже к разработчику, не к художнику.

## 5. Иконки интерфейса — сейчас эмодзи вместо своего стиля

Эмодзи выбраны как временная заглушка и не вписываются в деревянно-латунный стиль игры:

| Эмодзи сейчас | Где | Значение |
|---|---|---|
| 🐟 | TopBar, SocialPanel | садок / рыба |
| ⚡ | TopBar | энергия |
| 🔊 / 🔇 | TopBar, SettingsModal | звук вкл/выкл |
| ⚙ | TopBar | настройки |
| ☰ | TopBar | меню |
| 🥪 | RodDock | еда |
| 🧺 | RodDock | прикормка |
| 🏆 | RodDock, BaseModal | трофей |
| ➤ | SocialPanel | отправить сообщение |

Хорошо бы заменить на единый набор простых line/flat-иконок в латунном/деревянном стиле
(16×16 или 24×24, SVG предпочтительнее — не зависит от шрифта/платформы, в отличие от эмодзи).

## Приоритет, если делать по частям
1. **Спрайты 10 видов рыб** — самый заметный пробел геймплейно (сейчас улов визуально неотличим).
2. **Фоновые слои 3 локаций** — самый заметный пробел визуально (весь передний план — примитивы).
3. **Иконки категорий снастей** (14 наживок + 9 категорий снаряжения) — сильно улучшит магазин/инвентарь.
4. **Свой набор UI-иконок** вместо эмодзи.
5. Погодные текстуры, всплески, фавикон — финальная полировка.
