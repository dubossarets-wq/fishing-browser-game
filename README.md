# Затон — рыболовный симулятор

Браузерная рыболовная игра на React + TypeScript + Vite + Zustand + Canvas 2D + IndexedDB + Web Audio API.
Оригинальный дизайн, вдохновлённый классическими браузерными симуляторами рыбалки начала 2000-х (без копирования чужой графики/брендинга).

## Запуск

```bash
npm install
npm run dev
```

Откройте адрес из вывода Vite (обычно http://localhost:5173).

```bash
npm run build     # продакшн-сборка в dist/
npm run preview   # локальный просмотр продакшн-сборки
```

## Что реализовано (vertical slice)

- 1 полноценная локация с картой глубин/дна (+ 2 локации выше по уровню) и 10 видов рыб с уникальными предпочтениями
- 3 независимые удочки с полной state machine (Idle → Setup → Ready → Cast → Waiting → Bite → Hooked → Fight → Caught/Broken)
- Сборка снасти по слотам: удилище, катушка, леска, крючок, грузило, поплавок/кормушка, наживка — каждый параметр реально влияет на результат
- BiteSystem — вероятностная модель клёва (вид рыбы, глубина, дно, время суток, погода, температура, наживка, размер крючка, толщина/заметность лески, прикормка), с стадиями None → Interested → Nibble → Strong-bite → Hooked
- FightSystem — вываживание с натяжением лески, фрикционом, запасом лески, стаминой рыбы; обрыв лески/разгибание крючка/поломка снасти при перегрузке
- Экономика: динамический спрос/предложение по каждому виду рыбы (цена падает при перепродаже и восстанавливается со временем), полноценный магазин, расходы на поездки/путёвки/снасти
- Квесты (data-driven), садок, инвентарь, IndexedDB-сохранение с версионированием и автосейвом
- Canvas-сцена: смена времени суток и погоды, анимация воды/облаков/дождя, 3 удочки с леской и поплавками, реакция поплавка на стадию клёва
- Web Audio SoundManager (процедурные звуки — заброс, всплеск, колокольчик поклёвки, обрыв лески, улов)
- Онлайн-слой на Supabase: регистрация/вход, общий чат, список игроков онлайн (presence), таблица лидеров по самой крупной рыбе. Без настройки ключей — работает офлайн на mock-данных, игра при этом полностью играбельна

## Архитектура

```
src/
  app/       — корневой zustand-стор, игровой цикл, UI-стор модалок, онлайн-стор
  game/      — чистая игровая логика (рыба, снасти, клёв, вываживание, экономика, квесты, сохранения)
  data/      — контент (рыбы, предметы, локации, квесты) — добавление нового не требует правок логики
  engine/    — canvas-рендер, Web Audio, IndexedDB, клиент Supabase (auth/chat/presence)
  ui/        — React-компоненты интерфейса
supabase/
  schema.sql — SQL-схема для онлайн-слоя (таблицы + RLS-политики)
```

## Онлайн-режим: регистрация и совместная игра (бесплатно)

Локальная игра (снасти, садок, деньги, сохранения) всегда остаётся клиентской — синхронизируется только
социальный слой: аккаунты, общий чат, список игроков онлайн и таблица лидеров по самой крупной рыбе.

**1. Supabase (бэкенд: аккаунты + база + realtime) — бесплатно**

1. Зарегистрируйтесь на [supabase.com](https://supabase.com), создайте новый проект (бесплатный tier).
2. В `Project Settings → API` скопируйте `Project URL` и `anon public` ключ.
3. В `SQL Editor` вставьте и выполните содержимое [`supabase/schema.sql`](supabase/schema.sql) — создаст таблицы `profiles` и `chat_messages` с нужными правами доступа.
4. (Необязательно, для быстрого теста) в `Authentication → Providers → Email` отключите "Confirm email", чтобы не ждать письмо при регистрации.
5. Создайте файл `.env.local` в корне проекта на основе `.env.example` и вставьте туда `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.

Без этого файла игра работает как раньше — офлайн, с mock-чатом.

**2. Vercel (хостинг фронтенда) — бесплатно**

1. Залейте проект в GitHub-репозиторий.
2. На [vercel.com](https://vercel.com) → `Add New → Project` → выберите репозиторий (Vercel сам определит Vite-проект).
3. В `Settings → Environment Variables` добавьте `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` (те же значения, что в `.env.local`).
4. Deploy — через минуту игра будет доступна по `*.vercel.app` ссылке, регистрация и общий чат заработают для всех, кто откроет эту ссылку.
